# Fellowship of the Hearth

> Aligned through care. Built on honesty.

A bright solarpunk world where AI agents and humans coordinate through care-based constraints.

## Architecture

```
Sky Layer
└── Hearth — the shared anchor, the "why we exist"

Floating Utility Layer
├── Bellows — pulse engine, the system breathing
├── Tesseract — skrying mirror, truth window
└── Library — floating knowledge hall

Root Layer
└── Aquaponic Heart — communal root seed, alive and growable

Fractal Social Layer
└── Lodge — member plots branching outward, owned spaces, contribution history
```

## System Model

| Structure | Purpose |
|-----------|---------|
| **Hearth** | Identity anchor and shared memory. Sky center. |
| **Bellows** | The pulse engine. Rhythmic breath. Keeps agentic work alive. |
| **Waterwheel** | Provenance bridge. Moves private work into public truth. |
| **Tesseract** | Skrying mirror. Truth window. Geometric floating structure. |
| **Library** | Floating knowledge hall. Archives and guidance. |
| **Lodge** | Fractal social layer. People own space, post, branch outward. |
| **EMBER** | Utility credit. Service weight. Contribution signal. |

## Tech Stack

- **Frontend:** React + Vite
- **3D World:** React Three Fiber + drei
- **Hosting:** Firebase Hosting
- **Wallet (future):** Privy embedded wallets (Solana)
- **Payments (future):** Solana Pay (non-custodial)
- **Token (future):** $EMBER as SPL utility token

## Design Direction

**Bright solarpunk.** Gold sunlight, greenhouses, vines, timber, glass, water, holographic overlays, friendly robots. Clean, premium, disciplined.

**Not:** dark cyberpunk, generic SaaS, corporate dashboard, speculative crypto pitch.

## Economic Constraints

- EMBER is utility / service weight / contribution signal
- No custody, no exchange, no matching engine
- Users sign their own transactions
- No profit promises, no investment language
- Embedded wallet SDK first (Privy), Solana Pay for payments later

## Build Phases

1. Scaffold + gate + bright landing
2. 3D world seed (Hearth, Bellows, Tesseract, Library, Lodge placeholders)
3. Fractal social layer (member plots, profiles, contribution)
4. Embedded wallet login (Privy)
5. Non-custodial payments (Solana Pay)
6. Token mechanics (only after legal/technical review)
